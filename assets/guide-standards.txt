BOSS GUIDES — RESEARCH LOCK, CONTENT TEMPLATE, QA RUBRIC
========================================================
Do not edit this file for player-facing text; it is editor reference only.

RESEARCH LOCK (Cataclysm Classic)
--------------------------------
Primary: Official Dungeon Journal in-game (spell names, tooltips, phase order).
Secondary: Wowhead Cataclysm Classic raid / boss pages and spell database.
Tertiary: WoWpedia articles scoped to Cataclysm-era behavior.

HP and strict timers: treat as approximate; Classic tuning may differ by patch or server. Prefer mechanics and priorities over exact millions unless sourced.

CONTENT TEMPLATE (each *Guide.html)
-----------------------------------
1) Overview — kill condition, main wipe cause, one-sentence context.
2) Abilities / phases — journal-aligned names; tag [N], [H], [N/H].
3) Normal loop — pull through repeat cycle; who moves when.
4) Heroic-only — separate block; no H mechanics described as Normal-only.
5) Fail modes — short bullets.
6) Roles — tank / heal / DPS tied to real abilities.
7) Quick reference — table mechanic → action; must match body.
8) Callers / assignments — only where the fight needs it (bounces, runners, CD rotas).

Preserve existing section IDs used by sticky nav (#overview, #phases, #heroic, etc.) unless intentionally migrating anchors.

SPELL COMPANION PAGES (*-Spells.html)
--------------------------------------
Each *Guide.html may pair with a same-folder companion: replace the "-Guide" suffix with "-Spells" (e.g. BoT-Valiona-Theralion-Spells.html next to BoT-Valiona-Theralion-Guide.html).

- Boss guides: every distinct mech-badge label becomes a same-styled link to the companion file plus hash anchor, with the same visible text. Anchors are lowercase slugs: letters, digits, hyphens; apostrophes stripped (Finkle's mixture becomes finkles-mixture).
- Spell page: one spell-block article per ability with a short curated summary (paraphrase from Dungeon Journal / Wowhead Cata — do not paste full tooltips). Optional Wowhead link: https://www.wowhead.com/cata/spell={id}
- Editor metadata (optional wowheadSpellId + custom blurb): assets/spell-manifest.json keyed as "{posix path to *Guide.html}|{anchor}" e.g. "Bastion of Twilight/Valiona and Theralion/BoT-Valiona-Theralion-Guide.html|blackout"
- Generated index of companions: assets/spell-pages-manifest.json (regenerate with: python tools/generate_spell_companions.py)
- Bosses with no mech-badge markup get a minimal spell page stub only; no "Spells" sticky link until badges exist.

QA RUBRIC (before marking a guide done)
---------------------------------------
[ ] Capitalized mechanics match journal / Wowhead Cata or are labeled colloquial.
[ ] Causality correct (e.g. Spine: Burning Tendons after Nuclear Blast on plate, not “three corruptions = tendon”).
[ ] Dispels, interrupts, soaks have an assigned owner where relevant.
[ ] [N]/[H]/[N/H] pills consistent; no H-only effects in normal-only sentences.
[ ] HP/timers sourced or softened with disclaimer.
[ ] Quick ref matches overview and phases.
[ ] Nav anchors and prev/next links still valid.
[ ] After UI rollout: contrast readable on tables/callouts; prefers-reduced-motion respected; :focus-visible visible.
[ ] mech-badge links resolve to *-Spells.html#anchor; spell page blurbs match journal/Wowhead Cata for that encounter.
